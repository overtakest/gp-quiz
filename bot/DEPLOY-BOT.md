# 🤖 Бот 24/7 на Cloudflare Workers (бесплатно, без CLI)

Приложение уже живёт на GitHub Pages. Здесь — как поднять **процесс бота** (приветствие
по `/start`, кнопки, карточка прогресса), чтобы он работал всегда, без твоего ПК.

Файл с кодом: [`cloudflare-worker.js`](cloudflare-worker.js).

## Шаги

1. **Зайди** на [dash.cloudflare.com](https://dash.cloudflare.com) (бесплатная регистрация).
2. Слева **Workers & Pages** → **Create application** → **Create Worker**.
3. Дай имя (например `gp-quiz-bot`) → **Deploy** (создастся заготовка).
4. **Edit code** → удали пример, **вставь весь код** из `cloudflare-worker.js` → **Deploy**.
5. Открой у воркера **Settings → Variables and Secrets** и добавь:
   | Имя | Значение | Тип |
   |---|---|---|
   | `BOT_TOKEN` | токен из @BotFather | **Secret** (зашифрованный) |
   | `WEBAPP_URL` | `https://overtakest.github.io/gp-quiz/` | Text |
   | `WEBHOOK_SECRET` | любая случайная строка (напр. `gp-8f3k2x`) | Text *(необязательно)* |

   Сохрани и снова **Deploy**.
6. **Активируй вебхук:** открой в браузере
   `https://<имя-воркера>.<твой-поддомен>.workers.dev/setWebhook`
   Точный адрес воркера показан на его странице. Должно ответить `{"ok":true, ...}`.
7. Готово ✅ — открой бота, нажми `/start`. Работает 24/7.

## 🔒 Про токен
Ты присылал токен в чат. Рекомендую в @BotFather сделать **/revoke** для этого бота,
получить новый токен и вписать уже его в переменную `BOT_TOKEN` воркера (в код токен не
попадает — только в зашифрованную переменную).

## Локальная альтернатива (без облака)
Если облако не нужно — можно запускать поллинг-версию на своём ПК:
```bash
cd gp-quiz/bot
node bot.js        # работает, пока открыт терминал
```
⚠️ Одновременно должен работать **только один** способ: либо вебхук (Cloudflare), либо
`node bot.js` (polling). Если включишь вебхук, локальный `node bot.js` будет выдавать 409 —
это нормально, просто не запускай оба сразу.
